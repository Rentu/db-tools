# 分布式 mysql 工具

> 应用场景

分布式数据库，数据库实例落地到不同数据库上

在开发期间需要统一清档，删表，改表等

---

字段 | 字段说明 |  枚举
:---:|:---:|:---:
host| 主机地址|-
db| 数据库名称|-
user|用户|-
passwd|密码|-
method| 是否循环获取数据库句柄| 空 : 不循环； traverse：循环获取
after| 后续绑定操作，  譬如：清表后设置自增值，可根据程序自定义 |`[table => '表名'，'num' => '自增值']`
db_suffix| 数据库名称后缀| -
range| 如果循环，数据库后缀字符串| 根据业务自定义
skip| 跳过的数据表|-
action| 操作类型|-
rule| 规则| 自定义，目前需要改动程序
---

程序与本人所设计业务架构耦合性太强，后续会更新，用设计模式改写一下，提高可用性
